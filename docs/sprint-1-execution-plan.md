## 스프린트 목표

- 결제/구독/웹훅/법적 동의/운영로그의 최소 운영 안정성을 확보한다.

## QA 테스트 플랜 (필수)

### 1) 결제/구독
- 단건 결제 성공/실패 콜백 정상 처리
- 구독 빌링키 발급 실패 시 재시도 가능한 에러 응답
- 동일 `orderId` 재요청 시 중복 차단

### 2) 웹훅
- 동일 이벤트 2회 전송 시 상태 1회만 반영
- 비정상 payload 수신 시 로그 적재 및 오류 응답

### 3) 인증
- OAuth 취소/실패 시 사용자 메시지 확인
- 세션 만료 후 자동 갱신 실패 시 안전 로그아웃

### 4) 법적/운영
- 최신 정책 미동의 사용자 접근 제어
- 동의 이력 저장값(사용자/버전/시점) 검증

## 데일리 운영 규칙
- 매일 15분 스탠드업: 블로커, 우선순위 재조정
- 매일 종료 전: 주요 지표
  - 결제 성공률
  - 웹훅 처리 성공률
  - 신규 에러 건수
- 블로커 24시간 이상 지속 시 즉시 범위 축소 또는 대안 확정

## 리스크 및 대응
- 토스 샌드박스/운영 차이로 검증 지연
  - 대응: 샌드박스 케이스 리스트 사전 고정, Mock 병행
- 웹훅 순서 역전/중복 이벤트
  - 대응: 상태 전이 규칙과 idempotency 먼저 확정
- 비개발자 운영자 사용성 부족
  - 대응: 관리자 기능은 조회/기본조작 우선으로 단순화
