# 한국형 SaaS 보일러플레이트 v2.0 백로그 (Jira 스타일)

## 운영 원칙
- 우선순위: P0(출시 차단), P1(출시 직후 필수), P2(개선)
- 상태: Todo / In Progress / Done
- 각 티켓에는 DoD(완료 기준)와 테스트케이스를 반드시 포함

## 단계별 작업 기록 규칙
- 모든 작업 결과는 `docs/execution-log.md`에 "날짜/단계/티켓/결과/다음 액션" 형식으로 남긴다.
- 티켓 상태가 변경되면 같은 날 로그에 근거(테스트 통과, 배포 확인, 실패 원인)를 함께 기록한다.
- 스프린트 종료 시 `Done` 티켓 목록과 미완료 원인을 로그 마지막에 요약한다.
- 문서만 먼저 작성한 경우에도 "문서화 완료"로 기록하고 구현 전 상태를 `Todo`로 유지한다.
