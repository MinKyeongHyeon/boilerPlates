# 한국형 SaaS 보일러플레이트 PRD v2.0 (Production)

## 1. 제품 비전
- 목표: 비개발자도 한국 시장에서 실제 유료 SaaS를 안전하게 운영할 수 있는 운영 완성형 인프라 제공
- 가치 제안: "10분 런칭"을 넘어 "결제/구독/환불/법적 대응/장애 대응"까지 기본 제공
- 성공 기준: 셋업 완료율, 결제 성공률, 실패 결제 복구율, 월 유지율, CS 처리시간

## 2. 타겟 사용자
- 1순위: 코딩 지식이 거의 없는 1인 창업자/기획자
- 2순위: 빠른 MVP 출시가 필요한 소규모 팀(0~3명)

## 3. 핵심 문제
- 결제 성공 이후 상태 동기화(승인/취소/환불) 누락
- 정기결제 실패(만료/한도/분실) 자동 복구 정책 부재
- 운영자용 조회/수정/감사 로그 체계 부족
- 법적 문구는 있으나 동의 이력/버전 관리 미흡

## 4. 제품 범위 (In Scope)
- 소셜 로그인(카카오/네이버) + 세션 보안 강화
- 단건/구독 결제 + 웹훅 기반 상태 동기화
- 구독 운영: 플랜, 무료체험, 해지예약, 즉시해지
- 환불 운영: 전액/부분 환불, 사유 코드, 상태 추적
- 법적 운영: 약관/개인정보 버전관리 + 동의 이력 저장
- 최소 관리자 콘솔: 유저/결제/구독 검색 및 처리
- 관측성: 에러 추적, 알림, 운영 대시보드

## 5. 제외 범위 (Out of Scope)
- 엔터프라이즈급 복잡 권한 모델
- 해외 결제수단 동시 지원
- 회계/세무 자동 신고 시스템

## 6. 사용자 여정
- 신규 가입 → 소셜 로그인 → 플랜 선택 → 결제 → 권한 활성화
- 구독 실패 → 자동 재시도 → 사용자 알림 → 결제수단 변경
- 환불 요청 → 운영자 승인 → 환불 API → 상태 동기화
- 정책 개정 → 버전 업데이트 → 재동의 유도

## 7. 기능 요구사항

### 7.1 인증
- OAuth 로그인/로그아웃
- 세션 자동 갱신 및 만료 처리
- 계정 탈퇴 및 재가입 정책

### 7.2 결제/구독
- 단건 결제 생성/성공/실패 처리
- 빌링 인증, 빌링키 발급, 정기결제 과금
- 실패 재시도 정책(D+1, D+3, D+7)

### 7.3 웹훅
- 토스 웹훅 서명 검증
- 이벤트 idempotency 보장
- 중복 수신 방지 및 재처리 안전성

### 7.4 권한제어
- 플랜 상태 기반 기능 접근 제어
- 결제 실패/해지 시 권한 자동 회수

### 7.5 관리자
- 유저/결제/구독 조회
- 수동 결제 재시도/해지/환불
- 처리 이력 로그

### 7.6 법적 준수
- 약관/개인정보처리방침 버전 관리
- 동의 시점, 버전, 식별정보 기록

## 8. 비기능 요구사항 (NFR)
- 보안: 비밀키 서버 전용, RLS, Rate Limit, CSP/CSRF 정책
- 신뢰성: 결제/구독 API 99.9% 가용성 목표
- 성능: 핵심 랜딩 LCP 2.5초 이하(모바일 기준)
- 관측성: 구조화 로그, Sentry, 알림(슬랙/이메일)
- 복구: DB 백업/PITR 및 월 1회 복구 점검

## 9. 데이터 모델 최소 스코프
- users
- plans
- subscriptions
- payments
- billing_keys
- refunds
- webhook_events
- policy_versions
- policy_consents

## 10. 운영 정책
- 실패 결제 재시도 3회 + 단계별 알림
- 환불 가능 기간/사유 정책화
- 해지 정책(즉시/만료시점) 명확화

## 11. KPI
- Activation: 가입 후 첫 결제 전환율
- Revenue: MRR, 결제 성공률, ARPU
- Retention: 월 유지율, 해지율, 실패 복구율
- Ops: 웹훅 처리 성공률, MTTR, CS 처리시간

## 12. 수용 기준
- 핵심 결제 E2E 95% 이상 통과
- 웹훅 중복 수신 시 데이터 정합성 100%
- 환불/해지 처리 후 1분 내 상태 반영
- 정책 개정 시 동의 이력 누락률 0%
